# Embudo Planificación — valores, evolución y precio

**Versión:** 1.0 · **Junio 2026**  
**Alcance:** solo el componente **Planificación** (no Espejo, Alquimia, Radar ni otros mundos).  
**Principio:** cada peldaño vende una **capacidad nueva**; el precio acompaña la **maduración de necesidad**, no un catálogo de pantallas.

---

## 1. Tesis del embudo

SISTEMICAR está pensado como **componentes que resuelven mundos distintos**. Planificación es un mundo: **cerrar el día con conciencia medida**.

La mecánica de precio debe ser **familiarización progresiva**:

1. El usuario entiende *para qué sirve el día* (Base).
2. Gana confianza con **resultados medibles** (Operativo — primer upsell).
3. Escala a **orden mental, imprevistos y fe en proyectos** (Soberanía — segundo upsell, más avanzado).

> **Decisión estratégica:** el desglosador de **conquista** (flota CONQUISTA — unidades y ritmo) se vende **antes** que el de **enfoque** (flota ENFOQUE — ring y cupos).  
> No porque enfoque sea menos valioso, sino porque exige más capacidad de ordenar pensamientos, tolerar interrupciones y sostener bloques sin jefe externo.

## 3. Inventario de valores vendibles (Planificación)

Cada ítem incluye: **qué es**, **dolor**, **resultado**, **prueba en demo**, **peldaño**.

### 3.1 Anillo de conciencia

| Campo | Contenido |
|-------|-----------|
| Qué es | Reloj del día: conquista (morado), entropía (rojo), libre (gris). Visualiza si el tiempo vivido estuvo cubierto por misiones conscientes. |
| Dolor | "No sé en qué se me fue el día" |
| Resultado | Mapa honesto de cobertura consciente vs huecos |
| Prueba | Abrir planificación → anillo con arcos del día → señalar entropía en un segmento |
| Peldaño | **Base** |

| Campo | Contenido |

| Prueba | Crear segmento → lanzar 1 vehículo → cerrar → ver segmento "cubierto" |

## 4. Por qué Operativo antes que Soberanía

| Criterio | Operativo (peldaño 2) | Soberanía (peldaño 3) |
|----------|------------------------|------------------------|
| Claridad del resultado | Unidades, récord, ritmo | Bloques, ideas, imprevistos |
| Fricción cognitiva | Baja–media | Media–alta |
| ROI visible | Inmediato ("hice 12") | Acumulativo (pasos, fe, orden) |
| Perfil típico | Producción, estudio medible, delivery | Tiempo libre, mente acelerada, proyectos |
| Riesgo si se vende primero | Bajo — el usuario "siente" el método | Mayor — puede abrumar sin hábito de cierre |

**Conclusión:** Operativo **familiariza** con el método (cerrar, medir, PS, termo). Soberanía **profundiza** cuando ya confía en el sistema.

## 5. Preguntas de autodiagnóstico (copy para `/pagos`)

Usar en lugar de "Elige stack Estudiante o Producción":

1. **¿Tu día cierra con estructura?** → Si no: Base.
2. **¿Necesitas saber cuánto produces de verdad (unidades, ritmo, récord)?** → Sí: Base + Operativo.
3. **¿Tu problema es ideas sueltas, imprevistos y poco avance en sueños grandes?** → Sí: agrega Soberanía (idealmente después de habituar cierre medible).

**Atajo:** producción medible → peldaño 2. Mente acelerada + proyectos → peldaño 3. Ambos → sistema completo.
